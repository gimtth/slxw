# 手机新闻应用软件设计文档

（API 联网真实新闻方案 | 登录注册 | SQLite 本地存储 | Android Studio + Java + XML | 白色清爽 UI）

## 一、项目概述

本项目是基于 Android 原生开发的轻量化新闻 APP，使用聚合数据新闻头条 API 获取真实新闻数据，使用 SQLite 实现用户注册登录、密码修改和新闻收藏。项目面向课程设计演示，重点体现新闻分类展示、联网动态获取新闻、简洁美观界面和本地数据持久化。

## 二、项目基础信息

1. APP 名称：速览新闻
2. 开发工具：Android Studio
3. 开发语言：Java
4. 页面实现：XML 布局文件 + Java 业务逻辑
5. 本地数据库：SQLite
6. 网络请求：OkHttp
7. JSON 解析：Gson
8. 图片加载：Glide
9. 核心数据来源：聚合数据新闻头条 API
10. UI 风格：白色为主，蓝色作为强调色，页面清爽、留白适中、信息层级明确

## 三、核心功能模块

1. 用户模块：注册、登录、记住登录状态、修改密码、退出登录。
2. 新闻模块：首页推荐新闻、分类新闻、下拉刷新、新闻列表、新闻详情。
3. 收藏模块：收藏新闻、取消收藏、查看收藏列表、从收藏进入详情。
4. 异常提示：无网络、接口请求失败、无数据、登录信息错误等场景给出 Toast 或空状态提示。

## 四、底部导航设计

主界面采用 3 个底部导航标签，使用 Material `BottomNavigationView` 实现。

1. 首页：默认展示推荐新闻，即接口 `type=top`。
2. 分类：展示分类入口，点击分类后加载对应新闻列表。
3. 我的：展示用户信息、收藏入口、修改密码、退出登录。

底部导航只出现在登录后的主界面，登录、注册、启动页、详情页、收藏页、修改密码页不显示底部导航。

## 五、页面设计

### 1. 启动页

用于短暂展示 APP 名称和图标，同时读取本地登录状态。如果已登录则进入主界面，否则进入登录页。

### 2. 登录页

包含账号、密码输入框，登录按钮和注册入口。登录时校验 SQLite 中保存的用户信息，密码使用加密后的结果进行比对。

### 3. 注册页

包含账号、密码、确认密码输入框。注册时校验账号非空、密码非空、两次密码一致、账号未被注册。注册成功后保存用户数据并返回登录页。

### 4. 主界面

主界面承载底部导航，内部切换首页、分类、我的三个页面。整体采用白色背景，顶部标题栏简洁展示当前页面名称。

### 5. 首页新闻页

展示推荐新闻列表。每条新闻显示标题、来源、时间和缩略图。支持下拉刷新。点击新闻进入详情页。

### 6. 分类新闻页

顶部展示分类选项，下面展示对应分类新闻列表。分类名称和接口参数对应关系如下：

| 分类名称 | 接口 type |
| --- | --- |
| 推荐 | top |
| 国内 | guonei |
| 国际 | guoji |
| 娱乐 | yule |
| 体育 | tiyu |
| 科技 | keji |
| 财经 | caijing |
| 游戏 | youxi |
| 汽车 | qiche |
| 健康 | jiankang |

### 7. 我的页面

展示当前登录账号、收藏入口、修改密码入口和退出登录按钮。退出登录后清除本地登录状态并回到登录页。

### 8. 新闻详情页

新闻详情页优先调用新闻详情接口获取正文内容。接口返回的 `content` 是 HTML 字符串，页面使用 `WebView` 展示。由于部分图片地址可能以 `//` 开头，加载前需要补全为 `https://`。如果详情接口无内容，则提供“查看原文”入口，通过新闻 `url` 打开原网页。

详情页展示标题、来源、时间、正文内容，并提供收藏/取消收藏按钮。

### 9. 收藏列表页

展示当前用户收藏过的新闻。列表内容来自 SQLite，不依赖网络。每条收藏显示标题、分类、来源、时间和缩略图。支持取消收藏，点击可进入新闻详情页。

### 10. 修改密码页

输入旧密码、新密码、确认新密码。旧密码校验通过且两次新密码一致后，更新数据库中的加密密码。

## 六、SQLite 数据库设计

数据库只保存课程设计所需的用户数据和收藏数据，结构保持轻量，但要支持去重、详情跳转和收藏列表展示。

### 表 1：用户表 user

| 字段名 | 数据类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 用户 ID |
| username | TEXT | UNIQUE NOT NULL | 登录账号 |
| password_hash | TEXT | NOT NULL | 加密后的密码 |
| salt | TEXT | NOT NULL | 密码盐值 |
| create_time | TEXT | NOT NULL | 注册时间 |

密码不明文保存。注册时为每个用户生成随机盐值，将“密码 + 盐值”使用 SHA-256 计算摘要后保存到 `password_hash`。登录和修改密码时使用相同方式计算后再比对。

### 表 2：新闻收藏表 collect

| 字段名 | 数据类型 | 约束 | 说明 |
| --- | --- | --- | --- |
| id | INTEGER | PRIMARY KEY AUTOINCREMENT | 收藏 ID |
| username | TEXT | NOT NULL | 关联用户账号 |
| uniquekey | TEXT | NOT NULL | 新闻唯一 ID |
| news_title | TEXT | NOT NULL | 新闻标题 |
| news_url | TEXT |  | 新闻原文链接 |
| news_source | TEXT |  | 新闻来源 |
| news_date | TEXT |  | 新闻发布时间 |
| category | TEXT |  | 新闻分类 |
| thumbnail_url | TEXT |  | 新闻缩略图 |
| collect_time | TEXT | NOT NULL | 收藏时间 |

收藏表需要对 `username + uniquekey` 设置唯一约束，避免同一个用户重复收藏同一条新闻。

## 七、新闻 API 接口方案

### 1. 新闻列表接口

接口地址：`https://v.juhe.cn/toutiao/index`

常用参数：

| 参数 | 是否必填 | 说明 |
| --- | --- | --- |
| key | 是 | 聚合数据 AppKey |
| type | 否 | 新闻分类，默认 `top` |
| page | 否 | 页码，默认 1 |
| page_size | 否 | 每页条数，最大 30 |
| is_filter | 否 | 是否只返回有详情内容的新闻，建议传 1 |

列表页主要使用字段：`uniquekey`、`title`、`date`、`category`、`author_name`、`url`、`thumbnail_pic_s`、`is_content`。

### 2. 新闻详情接口

接口地址：`https://v.juhe.cn/toutiao/content`

常用参数：

| 参数 | 是否必填 | 说明 |
| --- | --- | --- |
| key | 是 | 聚合数据 AppKey |
| uniquekey | 是 | 新闻唯一 ID |

详情页主要使用字段：`content`、`detail.title`、`detail.date`、`detail.category`、`detail.author_name`、`detail.url`、`detail.thumbnail_pic_s`。

## 八、网络请求与数据解析

1. 使用 OkHttp 发起异步请求，避免在主线程访问网络。
2. 使用 Gson 将 JSON 转换为 Java 实体类。
3. UI 更新必须回到主线程执行。
4. 接口返回 `error_code=0` 时表示成功，否则展示错误提示。
5. 无网络、超时、接口额度不足、数据为空时均应给出友好提示。
6. Manifest 中需要添加网络权限：

```xml
<uses-permission android:name="android.permission.INTERNET" />
```

## 九、界面设计要求

1. 主背景使用白色，避免复杂背景。
2. 主题色使用清爽蓝色，按钮、选中状态和重点信息使用主题色。
3. 列表使用 `RecyclerView`，新闻卡片不做厚重阴影，保持轻量。
4. 图片圆角小、比例固定，避免列表跳动。
5. 文字层级清楚：标题深色，来源和时间浅灰色。
6. 空状态、加载中、错误状态使用简短文字提示。
7. APP 图标建议设计为蓝色圆角底 + 白色报纸/闪电/资讯线条图形，体现“快速浏览新闻”。如果不单独绘制，可使用 Android VectorDrawable 实现简单图标。

## 十、业务流程

1. 首次打开：启动页判断登录状态，未登录进入登录页。
2. 注册：输入账号和密码，密码加盐哈希后存入 SQLite。
3. 登录：读取用户盐值，计算密码哈希并比对，成功后保存登录状态。
4. 首页：进入主界面后自动请求推荐新闻。
5. 分类：点击分类后按对应 `type` 请求新闻列表。
6. 详情：点击新闻后进入详情页，请求详情接口并展示 HTML 正文。
7. 收藏：详情页点击收藏，将新闻关键字段写入收藏表。
8. 我的：可查看收藏、修改密码、退出登录。

## 十一、实验要求覆盖

1. 新闻分类：通过分类页和接口 `type` 参数实现。
2. 动态获取新闻：通过聚合数据 API 实时联网获取。
3. 界面简洁美观：白色清爽 UI、底部导航、列表布局。
4. 登录注册：SQLite 保存用户信息，密码加密存储。
5. 本地持久化：SQLite 保存用户和收藏数据。
6. 原生开发：Android Studio + Java + XML。

## 十二、技术亮点

1. 使用真实新闻 API，演示效果直观。
2. 密码加盐哈希存储，比明文保存更合理。
3. 收藏表保存完整新闻关键字段，离线也能展示收藏列表。
4. 使用 WebView 展示新闻 HTML 正文，兼容图文内容。
5. 底部导航结构清晰，符合常见新闻 APP 使用习惯。
