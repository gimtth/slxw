# 代码落地任务文档

## 阶段一：基础配置

1. 修改 APP 名称为“速览新闻”。
2. 添加网络权限 `INTERNET`。
3. 添加 OkHttp、Gson、Glide、RecyclerView、SwipeRefreshLayout 等依赖。
4. 准备主题色、文字颜色、通用间距和按钮样式。
5. 设计或替换 APP 图标，优先使用蓝色清爽风格的 VectorDrawable 图标。

## 阶段二：数据模型与工具类

1. 创建新闻列表实体、新闻详情实体、用户实体、收藏实体。
2. 创建 API 常量类，保存接口地址、分类参数和 AppKey。
3. 创建网络请求工具类，统一封装 OkHttp 请求。
4. 创建 Gson 解析逻辑，将接口 JSON 转为实体对象。
5. 创建密码工具类，实现随机盐值生成和 SHA-256 加盐哈希。

## 阶段三：SQLite 数据库

1. 创建数据库帮助类，管理建表和升级。
2. 创建 `user` 表，保存账号、密码哈希、盐值和注册时间。
3. 创建 `collect` 表，保存用户收藏的新闻关键字段。
4. 给 `username + uniquekey` 添加唯一约束，避免重复收藏。
5. 封装用户注册、登录校验、修改密码、收藏、取消收藏、查询收藏等方法。

## 阶段四：登录注册流程

1. 创建启动页，判断本地登录状态。
2. 创建登录页，完成账号密码校验和登录状态保存。
3. 创建注册页，完成账号唯一性校验、密码确认和加密存储。
4. 登录成功后进入主界面，退出登录后返回登录页。

## 阶段五：主界面与底部导航

1. 创建主界面 Activity。
2. 使用 BottomNavigationView 实现：首页、分类、我的。
3. 每个导航页面使用 Fragment 承载。
4. 保持白色背景和清爽布局，页面标题清晰。

## 阶段六：首页新闻列表

1. 创建新闻列表布局和 RecyclerView 适配器。
2. 首页默认请求 `type=top` 的新闻。
3. 展示标题、来源、时间和缩略图。
4. 增加下拉刷新。
5. 点击新闻条目进入详情页。

## 阶段七：分类新闻

1. 创建分类入口 UI。
2. 分类名称和接口 `type` 参数保持一一对应。
3. 点击分类后请求对应新闻列表。
4. 复用新闻列表适配器展示分类结果。

## 阶段八：新闻详情

1. 创建新闻详情页。
2. 根据 `uniquekey` 请求详情接口。
3. 使用 WebView 展示接口返回的 HTML 正文。
4. 处理 `//` 开头的图片地址，补全为 `https://`。
5. 增加收藏和取消收藏按钮。
6. 详情接口无正文时，提供查看原文入口。

## 阶段九：我的页面与收藏

1. 我的页面展示当前账号。
2. 添加收藏列表入口。
3. 添加修改密码入口。
4. 添加退出登录按钮。
5. 收藏列表从 SQLite 读取当前用户收藏数据。
6. 收藏列表支持取消收藏和进入详情页。

## 阶段十：修改密码与异常处理

1. 创建修改密码页。
2. 校验旧密码、新密码和确认密码。
3. 修改成功后更新密码哈希和盐值。
4. 完善无网络、接口失败、无数据、加载中等提示。
5. 统一 Toast、空状态和加载状态的显示方式。

## 阶段十一：界面收尾

1. 调整所有页面间距、字号、颜色和按钮样式。
2. 新闻列表保持信息密度适中，避免页面拥挤。
3. 统一顶部标题栏和返回按钮。
4. 统一空状态文案。
5. 确保整体风格以白色清爽为主，蓝色只作为重点色。
